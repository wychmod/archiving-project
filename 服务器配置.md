## 后端配置(已经在服务器上完成)
### 安装uwsgi
在虚拟环境安装uwsgi
```shell
conda activate ascvd_python

conda install uwsgi
```
### 启动uwsgi并查看(启动后端)
uwsgi --ini uwsgi.ini   # 启动uwsgi，输出：[uWSGI] getting INI configuration from uwsgi.ini 代表启动成功，之后会自动在当前目录新建 uwsgi.pid 和 uwsgi.log
uwsgi --stop uwsgi.pid  # 停止
uwsgi --reload uwsgi.pid  # 重启

### uwsgi.ini详解
```shell
[uwsgi]
# 配置和nginx连接的socket连接 
# 监听8102 端口，对应容器外的 6102
socket=0.0.0.0:8102
# 配置项目路径，项目的所在目录
chdir=/root/ascvdProject/ascvd/
# 配置wsgi接口模块文件路径,也就是wsgi.py这个文件所在的目录名
wsgi-file=/root/ascvdProject/ascvd/AscvdBackend/wsgi.py
# 配置启动的进程数
processes=4
# 配置每个进程的线程数
threads=2
# 配置启动管理主进程
master=True
# 配置存放主进程的进程号文件
pidfile=uwsgi.pid
# 配置dump日志记录
daemonize=uwsgi.log
```
## nginx配置（已完成）
### 安装nginx
### nginx配置
文件路径/etc/nginx/conf.d

react.conf
```shell
server {
        # 监听8103 端口，对应容器外的 6103
        listen 8103;
        server_name 222.31.135.49;

        location / {
                add_header 'Access-Control-Allow-Origin' '*';
                add_header 'Access-Control-Allow-Credentials' 'true';
                add_header 'Access-Control-Allow-Methods' 'GET, POST, OPTIONS';
                add_header 'Access-Control-Allow-Headers' 'DNT,X-CustomHeader,Keep-Alive,User-Agent,X-Requested-With,If-Modified-Since,Cache-Control,Content-Type';
                root /root/ascvdProject/ascvd/ascvdreact/build;
                index index.html index.htm;
                try_files $uri $uri/ /index.html;
        }
}

```

django.conf
```shell
server {
        # 监听8104 端口，对应容器外的 6104，react前端访问的应该是6104不是8104
        listen 8104;
        server_name 222.31.135.49;

        # 固定写法
        location  / {
                include uwsgi_params;
                uwsgi_pass 222.31.135.49:6102; # 这里就是uwsgi.ini配置文件中的地址 不是8102
        }

        location /static {
        # 指定静态文件存放的目录
        alias /root/ascvdProject/ascvd/static/;
        }
}
```
